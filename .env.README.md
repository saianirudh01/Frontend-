# Environment Configuration Files

This project uses environment variables to configure API endpoints for different environments.

## Files

- **`.env.local`** - For localhost development
- **`.env.dev`** - For development network (192.168.x.x or 172.16.x.x)

## Usage

### For Localhost Development
Use `.env.local` when:
- Running servers on your local machine (localhost)
- AI Server on port 5500
- News API on port 5174

### For Network Development
Use `.env.dev` when:
- Running servers on network IP (172.16.2.64)
- Testing on mobile devices
- Team development with shared servers

## Switching Environments

To switch between environments, rename the appropriate file:

```bash
# Use localhost
cp .env.local .env

# Use development network
cp .env.dev .env
```

## Configuration

**Current Endpoints:**

| Service | Localhost | Development Network |
|---------|-----------|---------------------|
| AI Server | http://localhost:5500 | http://172.16.2.64:5500 |
| News API | http://localhost:5174 | http://172.16.2.64:5174 |
| Auth API | http://localhost:5174 | http://172.16.2.64:5174 |

## Default Behavior

If no `.env` file is present, the app defaults to **development network** URLs (172.16.2.64).
